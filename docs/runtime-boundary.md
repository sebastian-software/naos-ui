# Runtime Boundary

`@iktia/runtime` is a tiny platform-helper package. It exists to avoid repeating
small browser API details in generated output, not to host component semantics.

## Current Inventory

| Helper | Responsibility | Boundary |
| --- | --- | --- |
| `createIktiaEvent(name, detail, options)` | Create a native `CustomEvent` with Iktia's default event flags. | Platform utility only; it does not know about components, state, effects, DOM diffing, or lifecycle. |

## Allowed Helpers

Runtime helpers may wrap low-level platform behavior when doing so keeps
generated code smaller and easier to audit:

* native event construction defaults;
* small DOM or hydration assertions that do not retain component state;
* scheduling helpers that do not own reactivity;
* serialization or parsing helpers for explicit compiler output contracts.

## Disallowed Helpers

Runtime helpers must not become a framework runtime. Do not add:

* virtual DOM or reconciliation;
* component lifecycle ownership;
* hook or signal runtime semantics;
* module graph loading or component registration policy;
* form, action, router, data-loading, or server behavior.

Generated components may import `@iktia/runtime` only for the allowed
platform-helper categories above. Component state, computed values, effects,
control flow, and list rendering stay generated by the compiler.

## Review Checklist

Before adding a runtime helper:

1. Prove the helper is a platform detail rather than component semantics.
2. Add focused runtime tests for defaults, overrides, and edge cases.
3. Add or update compiler output tests when generated code imports the helper.
4. Keep the public documentation clear that Iktia does not ship a framework
   runtime.
